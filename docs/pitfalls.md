# 踩坑记录

记录在实际运行中遇到的问题和解决方案。

---

## 坑 1: 429 限流

**现象**: Ralph 反复报错退出，每个 story 都失败

**原因**: `sleep 2` 太短，API 限流

**修复**:
- ralph.sh 中 `sleep 2` → `sleep 10`
- CLAUDE.md 中加 429 重试策略（等 30 秒再试）
- 子 Agent 调用之间保持 5 秒间隔

**教训**: 对于长时间运行的任务，宁可慢一点也不要被限流卡死。

---

## 坑 2: 迭代上限

**现象**: Ralph 跑完 50 轮后正常退出，但任务没完成

**原因**: ralph.sh 默认 `MAX_ITERATIONS=50`

**修复**: 设为 100 或更大

**教训**: 启动前计算好需要多少轮（story 数 + 重试余量）。

---

## 坑 3: 假时间戳

**现象**: timeline.json 里的时间全是编造的

**原因**: Claude 进程没有调用 `date` 命令，直接编了一个时间

**修复**: CLAUDE.md 加硬规则：
```
所有时间必须从系统获取，绝对不能自己编造！
date '+%Y-%m-%dT%H:%M:%S'
```

**教训**: AI Agent 会"偷懒"编造数据，关键数据必须强制从系统获取。

---

## 坑 4: heartbeat 时间戳单位

**现象**: watchdog 显示 "上次更新: -28632 秒前"

**原因**: 有时写毫秒（13位），有时写秒（10位），watchdog 用秒比较

**修复**: watchdog 自动检测位数：
```bash
if [ ${#ts} -gt 12 ]; then
  echo $((ts / 1000))  # 毫秒 → 秒
fi
```

**教训**: 涉及时间戳时，始终明确单位，或自动检测。

---

## 坑 5: JSON 编码损坏

**现象**: 中文标题显示为乱码

**原因**: Windows 下某个 Claude 进程写 JSON 时使用了错误的编码

**修复**:
1. notifier.js 用 TITLE_MAP 硬编码映射表兜底
2. 读取 JSON 时先 strip BOM

**教训**: 在 Windows 上处理中文时，始终指定 UTF-8 编码。

---

## 坑 6: 子 Agent 超时

**现象**: 大目录提取反复失败

**原因**: 文件数太多，子 Agent 30 分钟超时

**修复**:
1. 分片处理：每个子 Agent 只处理一个子目录
2. 增加超时到 60 分钟
3. 重试机制：失败后缩小范围重试

**教训**: 大任务必须分片，不能指望一个 Agent 一次搞定。

---

## 坑 7: PDF 中文乱码

**现象**: PDF 打开后中文全是方块

**原因**: ReportLab 默认用 Helvetica + WinAnsiEncoding，不支持中文

**修复**: 注册 CJK 字体（SimHei/SimSun）再生成 PDF

**教训**: 生成 PDF 时必须确保字体支持目标语言。

---

## 坑 8: 进程后台化

**现象**: 关闭终端后 Ralph 和 Watchdog 都停了

**原因**: 没有使用 `nohup`

**修复**:
```bash
nohup bash ralph.sh > ralph.log 2>&1 &
```

**教训**: 后台进程必须用 `nohup` + `&`。

---

## 通用建议

1. **先小规模测试**: 用 2-3 个简单 story 测试流程，再跑大批量
2. **日志要充分**: heartbeat/checkpoint/progress/timeline 四层日志缺一不可
3. **状态要持久化**: 所有状态存文件，不依赖进程内存
4. **超时要设置**: 子 Agent、API 调用、看门狗都要有超时
5. **编码要统一**: 全程 UTF-8，特别是 Windows 环境
